# Contributing to Selenium Boot

Thank you for your interest in contributing.

---

## Contribution Principles

- Follow existing architecture contracts
- Avoid adding flexibility without justification
- Prefer clarity over cleverness

---

## Pull Request Rules

- One concern per PR
- Documentation updates required for behavior changes
- Breaking changes must be justified

---

## What Will Be Rejected

- Hidden abstractions
- Global mutable state
- Framework behavior changes without discussion

---

## Summary

This project values discipline over speed.
